# 制备参数推荐系统 - 默认元需求与约束

本文档汇总了制备参数推荐系统的默认元需求和约束条件。这些是系统内置的默认行为，用户可以根据需要修改。

---

## 1. 实验室设备约束

### 1.1 温度约束

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **最高允许温度** | 1000℃ | 炉子封顶温度 |
| **最低降温速率** | 0.5℃/h | 设备能稳定控制的下限 |
| **最高降温速率** | 300℃/h | 设备控制上限 |
| **最长实验时长** | 200h | 超过此时间会提示"太久" |

### 1.2 降温速率约束

- **范围**：[0.5, 300] ℃/h
- **说明**：低于最小值无法精确控制，高于最大值可能损坏晶体

### 1.3 实验时长约束

- **最长单次实验时长**：200h
- **超过时提示**：建议缩短实验时间

## 2. 温度程序约束

### 2.1 元素沸点约束

| 约束类型 | 规则 | 适用场景 |
|----------|------|----------|
| **最高温段保温温度** | 可以超过制备元素中沸点最低的温度 | 用于熔化高沸点元素 |
| **次高温段保温温度** | 不得超过制备元素中沸点最低的温度 | 用于反应和晶体生长 |
| **温度层次原则** | 最高温段 > 次高温段 > 低温段 | 相邻段温差建议≥50℃ |

### 2.2 温度层次约束

- **规则**：Tmax（最高温段）> 次高温段 > 低温段
- **温差建议**：相邻段温差 ≥ 50℃

### 2.3 安全余量

- **温度安全余量**：5℃
- **说明**：在计算允许温度上限时，自动预留 5℃ 安全余量

## 3. 助熔剂约束

### 3.1 助熔剂选择原则

| 原则 | 说明 |
|------|------|
| **安全性** | 优先考虑安全稳定的助熔剂 |
| **化学惰性** | 避免与容器或原料发生剧烈反应 |
| **挥发性** | 避免高挥发性助熔剂导致成分变化 |

### 3.2 常见助熔剂推荐

| 助熔剂 | 适用场景 | 备注 |
|--------|---------|------|
| Sn | 通用 | 最常用的助熔剂 |
| Pb, Bi | 低熔点合金 | 适用于温度敏感材料 |
| Na2S, Na2Se | 硫族化合物 | 适用于 chalcogenide 材料 |

### 3.3 助熔剂比例参考

| 类型 | 比例范围 | 说明 |
|------|---------|------|
| 高稀释 | > 10 | 适用于易偏析材料 |
| 中等 | 2-10 | 通用范围 |
| 低 | < 2 | 自助熔剂体系 |

## 4. 容器与设备约束

### 4.1 容器选择

| 容器 | 适用场景 | 备注 |
|------|---------|------|
| Al₂O₃ 坩埚 | 通用 | 最常用的惰性容器 |
| 石英管 | 高温真空 | 适用于挥发性材料 |
| 不锈钢管 | 高压 | 适用于高压生长 |

### 4.2 分离方式约束

| 分离方式 | 适用场景 | 温度范围 |
|---------|---------|---------|
| 离心 | 小晶体批量分离 | 通常 600℃ |
| 酸洗 | 去除助熔剂残留 | 室温或加热 |
| 倾析 | 大晶体分离 | 通常 500-700℃ |
| 机械分离 | 热处理后 | 取决于材料 |

## 8. 如何修改这些约束

### 8.1 修改实验室约束

编辑 `data/recommand_window/input_template.jsonl`：

```json
{
  "实验室约束": {
    "最高允许温度_摄氏": 1200,    // 修改为你的设备上限
    "最长单次实验时长_h": 300      // 修改为你的实验周期
  }
}
```

### 8.2 修改元素沸点约束

编辑 `src/recommend_recipe.py` 中的 `build_baseline_scheme()` 函数：

```python
# 在 _min_boiling_point_from_target() 函数中添加新元素
boiling_points = {
    # ... 现有元素 ...
    '新元素': 1234.0  // 添加新元素的沸点
}
```

### 8.3 添加新的约束规则

参考 `add-constraint-sop.md` 文档，按照标准流程添加新的约束条件。

## 9. 约束优先级

当多个约束冲突时，系统按以下优先级处理：

1. **安全约束**（最高优先级）
   - 元素沸点约束
   - 设备温度上限

2. **物理可行性约束**
   - 温度层次原则（Tmax > 次高温 > 低温）
   - 相邻段温差 ≥ 50℃

3. **设备能力约束**
   - 降温速率范围
   - 实验时长限制

4. **用户偏好**（最低优先级）
   - 方案数量
   - 优化目标

## 10. 调试与验证

### 10.1 查看约束是否生效

运行 `--debug` 模式查看日志：

```bash
python runner/run_pipeline.py --query "xxx" --debug
```

### 10.2 检查输出

验证输出 JSON 中的约束字段：
- `约束验证` - 是否通过
- `风险来源简述` - 风险提示

### 10.3 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 温度被强制调整 | 超过元素沸点约束 | 检查沸点表配置 |
| 温差不足 | 相邻段温差 < 50℃ | 调整温度层次 |
| 时长警告 | 超过 200h | 缩短保温或降温时间 |
